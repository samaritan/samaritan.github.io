Dataset

The dataset curated as part of the samaritan project contains four contains four continuous-valued metrics collected from four popular open-source projects (Chromium, Firefox, Linux, and OpenBSD) spanning two domains (browser and operating system). In addition to the four continuous-valued metrics, the dataset also contains one boolean-valued metric which was collected only from the Chromium project.

Download

The dataset is available for download as a compressed archive. For backward compatibility, files containing individual metrics (churn, collaboration, contribution, lines of code, and offender) are available for download as a compressed archive as well.

Structure

A description of the structure of the dataset is presented in the table below.

Field Type Description
domain Factor The domain of the project. br for Browser and os for Operating System are the levels of the factor.
project Factor The name of the project. chromium, firefox, linux, and openbsd are levels of the factor.
path String The path of the source code file from which the metrics were collected. The path is relative to the root of the source code directory of the project.
churn Numeric Churn of a source code file is the total number of lines added and deleted throughout the life the file.
collaboration Numeric Collaboration of a source code file is the edge betweenness centrality of the edges in a developer collaboration network.
contribution Numeric Contribution of a source code file is the node betweenness centrality of the nodes in a developer contribution network.
loc Numeric Lines of Code (LoC) of a source code file is the total number of (source) lines of code in the file.
offender Logical Offender for a source code file is TRUE if the file was fixed for a vulnerability in the past, FALSE otherwise
